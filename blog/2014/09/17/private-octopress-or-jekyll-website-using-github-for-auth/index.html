Private Octopress (or Jekyll) Website Using GitHub for Auth

What to do when you have a private GitHub repo that you want privately publish some extra information for?

Examples include:

  • Octopress or Jekyll styled blog for developer notes and ideas
  • static code documentation, eg Doxygen
  • other code analysis
  • etc

That is, just about anything beyond an embedded readme.md file in your source code.

The short answer is to craft a custom application that uses the GitHub API to do OAUTH authentication.

An easy answer is to stand on the shoulders of giants and use a library: Jekyll Auth!

In hindsight this really is super easy, but there are a few points that I got stuck on.

Overview

Jekyll Auth is based on Sinatra.

Sinatra is a very easy to use DSL for creating web apps.

It’s written in Ruby.

Sinatra GitHub OAuth is included, and does the magic of checking with GitHub to see if a user is allowed access.

It also uses Jekyll to generate a static website from Markdown files.

It uses the Heroku web-hosting platform to generate and host the protected website.
